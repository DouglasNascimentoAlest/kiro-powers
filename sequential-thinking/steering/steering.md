# Resolver Problemas Complexos
- Inicie uma sequência de pensamento com o problema principal
- Divida em etapas menores e mais gerenciáveis
- Revise e ajuste o pensamento conforme necessário
- Marque como completo quando chegar à solução

# Análise de Arquitetura
- Use pensamento sequencial para avaliar trade-offs
- Documente cada consideração como um passo
- Permita ramificações para explorar alternativas
- Consolide conclusões no final

# Debugging
- Comece com a descrição do erro
- Adicione hipóteses como passos sequenciais
- Revise hipóteses baseado em evidências
- Documente a solução encontrada

# Planejamento de Features
- Inicie com requisitos de alto nível
- Quebre em tarefas menores
- Identifique dependências entre tarefas
- Ajuste o plano conforme descobertas
