# Testes E2E
- Sempre aguarde elementos antes de interagir
- Use seletores estáveis (data-testid, role, name)
- Capture screenshots como evidência
- Verifique estados após ações (redirecionamentos, mensagens)

# Web Scraping
- Use browser_snapshot para obter estrutura acessível
- Navegue página por página se necessário
- Respeite rate limits e robots.txt
- Extraia dados de forma estruturada

# Debugging de UI
- Capture screenshots em pontos chave
- Use browser_console para inspecionar estado
- Monitore requisições de rede para APIs
- Verifique erros no console do browser

# Automação de Formulários
- Preencha campos na ordem correta
- Aguarde validações antes de submeter
- Verifique mensagens de erro e sucesso
- Teste casos de borda (campos vazios, inválidos)
